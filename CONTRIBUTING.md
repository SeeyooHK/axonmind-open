# Contributing to Axonmind Open

Welcome! We appreciate your interest in improving **Axonmind Open**. To maintain high code quality and a smooth workflow, please follow these guidelines.

---

## 🛠️ How to Participate
 
 ### 1. Find or Create an Issue
 Check the [GitHub Issues](https://github.com/seeyooHK/axonmind-open/issues) to see if a topic is already being discussed.
 * **Reporting a bug:** Create a new issue describing the bug and how to reproduce it.
 * **Proposing a feature:** Open an issue to discuss your ideas.
 
 ---
 
 ## 🚀 Contribution Policy
 
 **We do not accept code contributions (pull requests) to this repository at this time.** This allows us to maintain clear intellectual property ownership of the codebase for Axonmind commercial distribution.
 
 ### How to contribute
 We still welcome and value community participation in other forms:
 * **Bug Reports**: Open an issue on GitHub describing the bug and how to reproduce it.
 * **Feature Requests**: Propose new features or improvements via GitHub Issues.
 * **Documentation**: Suggestions for documentation improvements are welcome via issues.
 
 If you have found a bug or have a suggestion, please open an issue and we will address it in a future release.
